# 入门指南

## 项目简介

NGA OH 是基于 HarmonyOS ArkTS 开发的 NGA 论坛客户端。

- **模型**：Stage 模型
- **API 版本**：6.1.0 (23)
- **模块**：单 entry 模块
- **构建系统**：Hvigor（DevEco Studio 内置工具链）

## 开发环境

### 前置条件

- 安装 DevEco Studio（内置 Hvigor 工具链）
- 设置 SDK 路径环境变量：

```bash
export DEVECO_SDK_HOME="C:/Program Files/Huawei/DevEco Studio/sdk"
```

### 构建命令

```bash
# 配置别名（推荐）
alias hvigorw="/c/Program\ Files/Huawei/DevEco\ Studio/tools/hvigor/bin/hvigorw.bat"

# 调试 HAP
hvigorw assembleHap --mode module -p module=entry@default -p buildMode=debug --no-daemon

# 发布 HAP
hvigorw assembleHap --mode module -p module=entry@default -p buildMode=release

# 清理
hvigorw clean
```

## 项目结构

```
entry/src/main/ets/
├── entryability/          # 应用生命周期入口（UIAbility）
├── entrybackupability/    # 备份恢复能力
├── pages/                 # 22 个页面级组件
├── common/                # 57 个可复用 UI 组件、工具类、常量
├── store/                 # 21 个状态管理文件
├── service/               # 10 个网络请求、外部服务调用文件
├── model/                 # 17 个数据模型定义
└── parser/                # 34 个数据解析器
```

## 核心依赖关系

- **数据流**：`parser/` → `service/NgaApi` ←→ `service/NgaClient` → 网络
- **状态流**：`store/` → `pages/` + `common/`（响应式绑定）
- **渲染流**：`model/` ← `parser/` ← `service/BBCodeParser` → `common/BBCodeContentView`
