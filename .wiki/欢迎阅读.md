# NGA OH 客户端 — 欢迎阅读

## 项目简介

NGA OH 是基于 HarmonyOS ArkTS 开发的 NGA 论坛客户端，采用 Stage 模型、API 6.1.0 (23)，单 entry 模块架构。通过 HTTP 协议与 NGA 官方 API 交互，支持论坛浏览、帖子阅读、回复、搜索、用户管理等核心功能，内置 BBCode 富文本渲染引擎与自适应多列布局。

## 模块依赖关系

> 文件数为各目录下 `.ets` 源文件实际数量（已对齐 17 项模块化重构后的真实目录结构）。

| 模块 | 目录 | 文件数 | 依赖方 |
|------|------|--------|--------|
| 页面组件 | `pages/` | 24（含 `ai/` 子目录 2 个） | common、store、service、model |
| 公共组件 | `common/` | 58 | store、model、service |
| 状态管理 | `store/` | 22 | service（PreferencesStore 仅本地持久化） |
| 服务层 | `service/` | 14（含 `api/` 7 个 + `ai/` 4 个） | parser、model、common |
| 数据模型 | `model/` | 19 | 无（纯类型定义） |
| 解析器 | `parser/` | 35（含 `md/` Markdown 解析 + `bbcode/` BBCode 解析） | model |
| 应用入口 | `entryability/` | 1 | store、common |
| 备份恢复 | `entrybackupability/` | 1 | store（依赖持久化数据） |

> 项目简介、环境搭建、构建命令等开发指引请参阅 [入门指南](入门指南.md)。
