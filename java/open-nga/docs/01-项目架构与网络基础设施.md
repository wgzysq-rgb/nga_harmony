## 1. 项目概述

Open-NGA 是一个第三方 NGA（艾泽拉斯国家地理）论坛 Android 客户端，采用 **MVP (Model-View-Presenter)** 架构模式。

### 1.1 技术栈

| 层次 | 技术选型 |
|------|---------|
| 网络通信 | Retrofit 2 + OkHttp 3 + RxJava 2 |
| JSON 解析 | fastjson (阿里巴巴) |
| 本地存储 | Room (SQLite) + SharedPreferences |
| HTML 渲染 | 自定义 WebView + lib_core HTML转换引擎 |
| 图片加载 | Glide |
| 异步任务 | RxJava 2 + AsyncTask (旧版) |

### 1.2 模块结构

```
open-nga/
├── lib_base_logger/     # 日志模块 (ILogger/Logger/DebugLogger/ReleaseLogger)
├── lib_cloud/           # 云服务模块 (Bugly崩溃上报 + 友盟统计)
├── lib_common/          # 通用工具库 (SharedPreferences工具/自定义控件/RxJava基类)
├── lib_core/            # 核心HTML渲染引擎 (BBCode解码/表情/图片/附件HTML构建)
├── nga_phone_base_3.0/  # 主应用模块 (API/数据模型/MVP层/UI)
└── docs/                # 本文档目录
```

### 2.2 域名选择机制

域名通过 `ForumUtils.getAvailableDomain()` 从 SharedPreferences 读取用户选择的域名索引：

```java
// ForumUtils.java:15
public static String getAvailableDomain() {
    int index = Integer.parseInt(sp.getString(PreferenceKey.KEY_NGA_DOMAIN, "1"));
    return context.getResources().getStringArray(R.array.nga_domain)[index];
}
```

`Utils.getNGAHost()` 在域名后追加 `/`，返回格式如 `https://bbs.ngacn.cc/`。

## 3. 网络层架构

### 3.1 核心类关系图

```
RetrofitHelper (单例, 工厂)
  ├── 创建 OkHttpClient (含3个拦截器)
  ├── 创建 Retrofit 实例
  │     ├── JsonStringConvertFactory (GBK响应解码)
  │     └── RxJava2CallAdapterFactory
  └── 提供 RetrofitService 代理实例

RetrofitService (Retrofit接口)
  ├── get() / getByForum() — GET请求
  ├── post() — POST表单请求
  ├── login() — 登录请求 (FormBody/MultipartBody)
  ├── getAuthCodeImage() — 获取验证码图片
  ├── uploadFile() — 文件上传
  └── get(url, headers) — 带自定义头的GET请求

HttpPostClient (原生HttpURLConnection)
  └── post_body(body) — POST请求 (用于发帖/私信/评论)

HttpUtil (原生HttpURLConnection)
  └── getHtml(uri, cookie) — GET请求 (旧版,用于获取版面列表等)
```

### 3.2 RetrofitHelper — 网络客户端工厂

**源码位置**: `nga_phone_base_3.0/src/main/java/sp/phone/http/retrofit/RetrofitHelper.java`

**职责**: 单例模式，管理 OkHttpClient 和 Retrofit 实例。

**默认 Base URL**: 从 SharedPreferences 动态获取用户选择的域名索引（默认索引为1，即 `https://bbs.ngacn.cc/`）。`https://bbs.ngacn.cc/` 仅用于静态方法 `getAuthCodeService()` 和 `getDefault()`

#### 3.2.1 OkHttpClient 拦截器链

**拦截器 1 — Cookie 与 User-Agent 注入**

```
位置: RetrofitHelper.java:98-112
触发: 所有请求
行为:
  1. 若请求未携带 Cookie 头，从 UserManagerImpl 获取当前用户的 Cookie

**拦截器 3 — 调试数据收集**

```

### 3.4 JsonStringConvertFactory — 响应转换器

**源码位置**: `nga_phone_base_3.0/src/main/java/sp/phone/http/retrofit/converter/JsonStringConvertFactory.java`

**职责**: 将 OkHttp `ResponseBody` 转换为 GBK 编码的 `String`。

```
输入: ResponseBody (字节流)
处理: IOUtils.toString(inputStream, "GBK")
输出: String (GBK解码后的JSON字符串)
副作用: 调用 Debugger.collectBody() 收集响应用于调试
```

### 3.5 HttpPostClient — 原生 POST 客户端

**源码位置**: `nga_phone_base_3.0/src/main/java/sp/phone/param/HttpPostClient.java`

**使用场景**: 发帖、发私信、发评论（不通过 Retrofit，直接使用 HttpURLConnection）

```
请求头:
  Cookie: {cookie}                    (如果设置)
  User-Agent: {WebView默认UA}         (从RetrofitHelper获取)
  Content-Type: application/x-www-form-urlencoded
  Content-Length: {body长度}
  Accept-Charset: GBK

请求方法: POST
请求体: URL编码的表单字符串
```

### 3.6 HttpUtil — 原生 GET 客户端

**源码位置**: `nga_phone_base_3.0/src/main/java/sp/phone/util/HttpUtil.java`

**User-Agent 格式**:
```
Nga_Official/573({Manufacturer} {Model};Android{Version})
示例: Nga_Official/573(Xiaomi MI 2S;Android4.1.1)
```

**请求头**:
```
Cookie: {cookie}
User-Agent: Nga_Official/573(...)
Accept-Charset: GBK
Connection: close
Accept-Encoding: gzip,deflate
```

**超时设置**: 连接 5000ms, 读取 10000ms  
**编码处理**: 自动检测 Content-Type 中的 charset，默认 GBK；支持 GZIP 解压

## 4. 认证与 Cookie 管理

### 4.1 认证机制

Open-NGA 使用 **Cookie 认证**，不使用 OAuth 或 Token 机制。

**Cookie 格式**:
```
ngaPassportUid={userId}; ngaPassportCid={cid}
```

- `ngaPassportUid`: 用户ID (字符串)
- `ngaPassportCid`: 用户会话凭证 (字符串)

**Cookie 生成**: `UserManagerImpl.getCookie()` → `"ngaPassportUid=" + uid + "; ngaPassportCid=" + cid`

### 4.2 用户数据存储

**Room 数据库表** (`app_database.db`, 版本3):

```sql
CREATE TABLE users (
    uid TEXT PRIMARY KEY,
    cid TEXT,
    nick_name TEXT,
    avatar_url TEXT
);

CREATE TABLE notes (
    note TEXT,
    nickName TEXT,
    userId TEXT PRIMARY KEY
);
```

**SharedPreferences 存储**:
- 用户激活索引 (`USER_ACTIVE_INDEX`)
- 黑名单 (`BLACK_LIST`, JSON数组)
- 头像URL (`PREFERENCE_AVATAR`, 独立SharedPreferences文件)

### 4.3 Cookie 注入流程

```
1. Retrofit 拦截器检查请求头是否已有 Cookie
2. 若无, 从 UserManagerImpl 获取当前活跃用户
3. 构造 "ngaPassportUid={uid}; ngaPassportCid={cid}" 字符串
4. 注入到请求头 Cookie 字段
5. 同时注入 User-Agent 和 X-User-Agent 头
```

## 5. 通用响应格式

### 5.1 JSON 响应前缀

大部分 API 响应以 `window.script_muti_get_var_store=` 为前缀，解析时需先移除：
```java
js = js.replace("window.script_muti_get_var_store=", "");
```

### 5.2 成功响应结构

```json
{
  "data": {
    "0": "操作结果消息"
  },
  "time": 1601530856
}
```

### 5.3 错误响应结构

```json
{
  "error": {
    "0": "错误描述信息"
  },
  "time": 1601530856
}
```

### 5.4 编码

所有 API 响应使用 **GBK** 编码（通过 JsonStringConvertFactory 处理）。
